Przeplyw danych

Plugin Monitor zbiera dane z platform e-commerce (wtyczki platnosci, paywall) i prezentuje je w centralnym dashboardzie. Integracja polega na wysylaniu eventow HTTP do API Plugin Monitor.

Paywall / Wtyczka
→
POST /api/ingest/batch
→
BullMQ Worker
→
PostgreSQL
→
Dashboard

Eventy sa klasyfikowane automatycznie:

  • payment_* i refund_* → zakladka Paywall Stats
  • error, warning, info, heartbeat → zakladka Plugin Errors

Wymagania wstepne

  1. Plugin Secret — wspolny sekret wbudowany w kazda wtyczke/integracje. Ustawiany jako PLUGIN_SECRET env var po stronie monitora. Merchant nie konfiguruje niczego.
  2. Monitor URL — hardcoded w kodzie wtyczki, np. https://monitor.planetpay.pl
  3. Auto-rejestracja — sklepy sa tworzone automatycznie przy pierwszym kontakcie (upsert wg domeny). Brak recznej rejestracji.

Autentykacja

Kazdy request wymaga headera X-Plugin-Secret:

X-Plugin-Secret: pm_shared_secret_change_in_production
Shared secret — nie API key per sklep Jeden wspolny sekret dla wszystkich instalacji wtyczki. Sklep jest identyfikowany po domenie (shopDomain w payload), nie po kluczu. Jesli domena jest nowa — shop tworzony automatycznie w bazie.
Merchant nic nie konfiguruje URL monitora i secret sa wbudowane w kod wtyczki. Nie ma zadnych ustawien dla merchanta — monitor dziala automatycznie z kazdej zainstalowanej wtyczki.

Rate limit: 60 requestow na minute per IP.

Eventy paywall (statystyki transakcji)

Te eventy trafiaja do zakladki Paywall Stats w dashboardzie.

typeKiedy wysylacWymagane metadata
payment_initiated Klient rozpoczyna platnosc (klik "Zaplac") orderId, method, amount, currency
payment_completed Platnosc zakonczona sukcesem (status COMPLETED lub SETTLED) orderId, transactionId, method, amount, currency, status
payment_failed Platnosc odrzucona/nieudana (REJECTED, CANCELLED, ERROR) orderId, status, rejectCode?, rejectInfo?
refund_initiated Refund zlecony orderId, amount, reason?
refund_completed Refund zrealizowany orderId, refundId, amount, status

Strategia buforowania i wysylania

Zalecana implementacja (analogicznie do wtyczki WooCommerce):

  1. Buforuj eventy lokalnie — nie wysylaj kazdego eventu osobno. Zbieraj w buforze (max 50 eventow).
  2. Flushuj przy naturalnych zdarzeniach — np. po otrzymaniu notyfikacji z PlanetPay, przy wejsciu admina.
  3. Throttle — max 1 flush co 5 minut (z wyjatkiem eventow krytycznych jak webhook platnosci).
  4. Non-blocking — wysylka w tle, timeout max 2-3s, nie blokuj glownego procesu.
  5. Fire & forget — nie czekaj na odpowiedz, nie ponawiaj w przypadku bledu.
  6. Ring buffer — jesli bufor pelny (50 eventow), wyrzuc najstarsze.

Triggery flush

TriggerTyp flushu
Otrzymanie notyfikacji payment z PlanetPay Natychmiastowy (flushNow)
Otrzymanie notyfikacji refund z PlanetPay Natychmiastowy (flushNow)
Wejscie admina do panelu Throttled (co 5 min)
Blad API PlanetPay Throttled (co 5 min)
Aktywacja / pierwszy start Natychmiastowy

Dobre praktyki

Nie wysylaj danych wrazliwych Nie wysylaj numerow kart, danych osobowych klientow (imie, email, adres). Wysylaj tylko: kwoty, metody platnosci, statusy, ID zamowien/transakcji.
Uzywaj UTC Wszystkie timestampy w formacie ISO 8601 z UTC (Z suffix), np. 2026-07-02T10:30:00Z.
Domena bez www shopDomain powinien byc czysty: sklep.pl, nie www.sklep.pl ani https://sklep.pl.
Fire & forget Jesli Plugin Monitor nie odpowiada — pomin. Eventy nie sa krytyczne. Brak ich nie wplywa na dzialanie paywall. Loguj bledy lokalnie (console.error), ale nie rzucaj wyjatkow.

Przyklady curl

Wyslij batch eventow

# Wyslij batch eventow
curl -X POST https://monitor.example.com/api/ingest/batch \
  -H "Content-Type: application/json" \
  -H "X-Plugin-Secret: pm_shared_secret_change_in_production" \
  -d '{
    "shopDomain": "sklep.pl",
    "pluginVersion": "1.0.0",
    "platform": "paywall",
    "events": [
      {
        "type": "payment_completed",
        "source": "PaywallNotificationHandler",
        "message": "Payment COMPLETED for order #1234",
        "metadata": {
          "orderId": "1234",
          "transactionId": "pp_txn_abc",
          "method": "BLIK",
          "amount": 149.99,
          "currency": "PLN",
          "status": "COMPLETED"
        },
        "occurredAt": "2026-07-02T10:30:00Z"
      },
      {
        "type": "error",
        "source": "PlanetPayClient",
        "message": "Timeout connecting to PlanetPay API",
        "stackTrace": "Error: ETIMEDOUT\n    at ...",
        "metadata": {
          "endpoint": "/v1/ecommerce/payment",
          "timeout": 10000
        },
        "occurredAt": "2026-07-02T10:29:55Z"
      }
    ]
  }'bash

Odpowiedz:

// 202 Accepted
{ "accepted": 2 }JSON
